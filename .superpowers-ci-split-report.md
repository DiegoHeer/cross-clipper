## Check names preserved

## Cross-workflow dependency dropped

The original `e2e` job had `needs: server`. `needs:` does not work across workflow files. The dependency is dropped; `server` and `e2e` now run in parallel (faster). Noted in an inline comment in `e2e.yml`.

## Paths-filter decisions

| Workflow | Paths filter | Reason |
|----------|-------------|--------|
| `server.yml` | **None** | "Server (lint + test)" is a required status check. With classic branch protection, a required check that never runs blocks merging. Must trigger on every PR. |
| `e2e.yml` | `server/**`, `.github/workflows/e2e.yml` | Not a required check; safe to skip on docs-only or packages-only changes. |
| `core.yml` | `packages/**`, `package.json`, `package-lock.json`, `.github/workflows/core.yml` | Not a required check; safe to skip on server-only or docs-only changes. |
