### Fixture design decisions

- **Session-scoped server**: booted once for journeys 1, 2, 4. Uses `_free_port()` (OS-assigned), `CC_ALLOW_REGISTRATION=false` (production default). Waits up to 30 × 200 ms on `/health`.
- **Single shared user**: all journeys on the session server use `owner@example.com`. Journey 1 performs the one allowed registration; journeys 2 and 4 log in with distinct device names to get fresh tokens and device IDs. This matches the intended single-user/multi-device model and avoids needing `allow_registration=true`.
- **Function-scoped `restart_server`**: journey 5 gets its own server so it can SIGTERM without affecting the session server. It registers its own user (fresh data dir).
- **`_stop_server`**: terminate → wait(5 s) → kill fallback. Prevents zombie processes.

## Concerns / known gaps

1. **Journey 3 (live WS events)** not implemented — arrives with the WS journeys PR per spec.
2. **Journey 4 WS-close half** not implemented — marked with `# TODO` comment in the test.
3. **Test ordering**: journeys 2 and 4 implicitly depend on journey 1 having registered the user. If pytest ever runs tests_e2e in isolation without journey 1, they will fail with 401. A session-scoped `ensure_user` autouse fixture could eliminate this brittleness — deferred for now; all four journeys pass in order.
4. **Port reuse in journey 5**: `time.sleep(0.5)` after SIGTERM to allow OS to free the port. Flaky on heavily loaded CI hosts. A retry loop on bind would be more robust.
5. **`asgi.py` import-time Settings**: `create_app()` is called at module import time, so env vars must be set before uvicorn starts (which they are in `_start_server`). No lazy factory needed.

### Finding 2 — SO_REUSEADDR removed from `_wait_port_free`

**Problem**: `s.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)` before `s.bind()` caused the check to succeed while the port was still in TIME_WAIT.  uvicorn binds without `SO_REUSEADDR`, so the probe was a false positive.

**Fix** (`server/tests_e2e/conftest.py`): removed the `setsockopt` call; plain `s.bind()` now mirrors what uvicorn sees.  Error message extended to mention TIME_WAIT.

### Finding 3 — durability assertion already present

The full-pull "no items lost" assertion (lines 326-334 of `test_journeys.py`) was present in the **initial** commit (`1289895`).  After restart the test:
1. Does a cursor=0 pull and checks each pre-kill `item_id` is in `recovered_ids`.
2. Asserts `len(recovered_ids) == len(set(recovered_ids))` (no duplicates).

No code change needed for finding 3.
